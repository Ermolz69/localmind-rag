# LocalMind Release - 2026-06-22

This release page is the canonical summary for the current LocalMind release state on `master` after the final stabilization merges through `#96`.

The release is focused on one message: LocalMind is now a defendable local-first desktop knowledge application. It can start the desktop shell, supervise the local backend, organize documents into buckets, ingest and index local files, preview documents, search local content, use RAG chat when the AI runtime is ready, expose diagnostics, and demonstrate Companion Mode as a phone-based remote interface over the local network.

## Executive summary

LocalMind now presents a cohesive local knowledge workflow:

1. The Tauri desktop app starts and supervises `KnowledgeApp.LocalApi`.
2. The React UI calls LocalApi through the shared API client only.
3. Documents are stored locally, indexed through durable ingestion jobs, and made available for search and RAG.
4. Notes, buckets, diagnostics, document preview, and runtime state are visible through the desktop UI.
5. Companion Mode extends the workflow to a trusted phone on the same LAN without making cloud services mandatory.
6. Documentation, generated API references, demo instructions, and release notes are published through the DocFX site.

## What changed

### Companion Mode v2

Companion Mode is the largest user-facing addition in this release. The phone can now act as a practical remote surface for the desktop LocalMind app after pairing on the local network.

Key changes:

- Added a LAN gateway listener inside LocalApi for Companion Mode while keeping the normal LocalApi desktop boundary loopback-first.
- Added durable trusted-device storage with hashed device tokens.
- Added per-device permissions and route allowlisting for companion access.
- Added QR pairing and reconnect behavior.
- Added phone screens for chat, search, documents, files, activity, and connection states.
- Added live activity feed updates for document state changes.
- Added allowed-folder navigation for phone-side file adding.
- Added five-state file-add lifecycle display on the phone.
- Added visible desktop copy explaining allowed and permanently blocked companion capabilities.

User impact:

- A user can pair a phone once, then browse local knowledge, search, add files, and monitor activity from the phone.
- Companion access is explicit, opt-in, permission-scoped, and limited to safe route categories.
- The desktop app remains local-first; remote services are not required for this workflow.

### Document preview and diagnostics

Document handling is now more demo-ready and easier to explain.

Key changes:

- Added document preview support, including an in-process OOXML-to-HTML fallback for supported Office formats.
- Improved diagnostics settings and log maintenance flows.
- Added path picker and runtime path visibility improvements.
- Added safer LocalApi process-tree lifecycle handling on Windows through a Job Object.
- Improved desktop runtime status and startup-state feedback.

User impact:

- Users can inspect documents without leaving LocalMind.
- Demo operators can explain where runtime data and logs live.
- Startup and shutdown behavior is more predictable on Windows.

### Buckets, documents, and context menus

The main workspace flows were tightened for defense use.

Key changes:

- Added right-click context menus for bucket cards and document rows.
- Moved destructive bucket/document actions behind context menus where appropriate.
- Added document deletion from both the global documents list and bucket details page.
- Fixed bucket `syncStatus` display so it renders meaningful labels instead of raw enum values.
- Preserved bucket descriptions during rename requests.
- Removed an unused bucket delete flag that caused the frontend check to fail.

User impact:

- Buckets and documents now behave more like a desktop application.
- Dangerous actions are still available, but less visually noisy.
- The frontend check is clean for the current bucket page implementation.

### Search, RAG, and indexing quality

The retrieval/indexing pipeline received a major stability pass.

Key changes:

- Replaced the earlier fixed character chunking baseline with `structure-aware-token-v3` as the default chunking strategy.
- Added token-aware forced splitting for oversized blocks.
- Added overlap only for forced token-window splits.
- Added heading, paragraph, source-span, page-number, and text-hash oriented metadata where available.
- Added embedding batching with validation of provider response count and ordering.
- Added diagnostics around indexing and chunk processing.
- Added tests for the tokenizer service and structure-aware chunker behavior.
- Updated retrieval/indexing architecture documentation to match current behavior.

User impact:

- Large extracted text blocks are less likely to produce oversized chunks.
- Search and RAG context are built from more controlled, explainable chunks.
- Indexing behavior is easier to debug when a document behaves unexpectedly.

The notes workspace is now part of the release narrative instead of a side panel.

Key changes:

User impact:

## Known boundaries

This release intentionally does not claim:

- production-ready cloud sync;
- internet relay access for Companion Mode;
- full mobile application packaging;
- signed installer and auto-update;
- OCR for scanned documents;
- production-scale vector indexing;
- guaranteed AI runtime availability on every machine.

These are follow-up release tracks. They do not block the local-first MVP story.

## Demo path

Recommended final demo order:

1. Open the docs site and show the release page.
2. Open the demo gallery from the `Instructions` tab.
3. Start the desktop app.
4. Show LocalApi readiness and runtime state.
5. Create or select a bucket.
6. Upload a document and show ingestion lifecycle.
7. Preview the document.
8. Search or chat with local content when the runtime is ready.
9. Show notes as a local workspace.
10. Show Companion Mode as the phone remote workflow.
11. Close with CI, docs, OpenAPI, and local-first architecture.
